# After 30 Med Bot — Telegram бот для напоминаний о приёме лекарств

> ⚡ Вайб-кодинг проект — написан в паре с Claude (AI).
> Код живой, рабочий, итерируем дальше.

## Стек

- Python 3.14
- python-telegram-bot 22.7 — Telegram API (async)
- APScheduler 3.11.2 — планировщик напоминаний (каждую минуту)
- SQLite — база данных
- pytz + timezonefinder + geopy — определение часового пояса
- fpdf2 + DejaVuSans TTF — генерация PDF

## Структура проекта

```
med-bot/
├── bot.py              # точка входа — регистрация handlers, main()
├── database.py         # вся работа с SQLite (CRUD + миграции)
├── scheduler.py        # отправка напоминаний каждую минуту
├── constants.py        # состояния диалогов, метки, слоты времени
├── utils.py            # общие утилиты (декоратор ошибок, escape_md, cancel)
├── broadcast.py        # скрипт рассылки сообщений всем пользователям
├── handlers/
│   ├── meds.py         # добавление, редактирование, удаление лекарств
│   ├── stats.py        # статистика за 7 дней, план на 7 дней
│   ├── export.py       # экспорт истории и плана в PDF
│   ├── settings.py     # настройки, пресеты времени, режим напоминаний
│   ├── admin.py        # админ-панель (только для ADMIN_ID)
│   └── timezone.py     # /start, определение часового пояса, главное меню
├── .env                # токен бота (не коммитить!)
├── requirements.txt
└── med_bot.db          # SQLite база (создаётся автоматически)
```

## База данных

| Таблица | Описание |
|---|---|
| `users` | пользователи: telegram_id, timezone, reminder_mode, пресеты времени, план дня |
| `medications` | лекарства: название, дозировка, способ приёма, кол-во приёмов |
| `schedule_rules` | правила расписания: время, тип, `dosage` (NULL = берётся из medications) |
| `intake_log` | история приёмов (taken / skipped / pending) |

### Типы расписания

| Тип | Описание |
|---|---|
| `daily` | каждый день |
| `interval` | каждые N дней от даты начала |
| `weekdays` | по выбранным дням недели (Пн–Вс) |
| `monthly` | раз в месяц, N-го числа |

## Команды бота

| Команда | Описание |
|---|---|
| `/start` | регистрация, запрос часового пояса, главное меню |
| `/meds` | список лекарств + добавить / изменить / удалить |
| `/stats` | история за 7 дней, план на 7 дней |
| `/settings` | настройки: часовой пояс, режим напоминаний, время приёмов |
| `/about` | о проекте |
| `/cancel` | отмена текущего диалога |

## Функционал

- **Лекарства на сегодня** — кнопка в главном меню, показывает расписание с отметками ✅ принято / ❌ пропущено / ⏳ ожидает
- **Добавление лекарств** с дозировкой, способом приёма, гибким расписанием
- **Multi-select время приёма** — выбирай 🌅 Утро / ☀️ Обед / 🌇 Вечер / 🌙 Ночь; любая комбинация
- **Персональные пресеты** — каждый пользователь настраивает своё время слотов в /settings
- **Гибкое расписание** — каждый день / через N дней / по дням недели / раз в месяц
- **Разная дозировка** — одно лекарство с двумя дозировками и отдельным расписанием для каждой (например: 25 мкг по нечётным / 50 мкг по чётным)
- **Напоминания** по расписанию с кнопками ✅ Принял / ❌ Пропустить
- **Два режима напоминаний**: один раз или повтор каждые 5 минут до подтверждения (до 2 часов)
- **План на день** — утреннее сообщение со списком лекарств на сегодня (настраивается в /settings)
- **История за 7 дней** с детализацией по дням, времени и статусу приёма
- **План на 7 дней** — показывает все лекарства по датам с учётом типа расписания
- **Экспорт в PDF** — история за 7 дней и план на 7 дней скачиваются кнопкой 📄 Скачать PDF; кириллица через DejaVuSans TTF
- **Определение часового пояса** по геолокации или городу
- **Поддержка нескольких пользователей**, лимит 10 лекарств на пользователя
- **Удаление данных** — пользователь может полностью удалить свои данные из /settings
